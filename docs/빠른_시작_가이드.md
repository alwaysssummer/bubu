# 🚀 빠른 시작 가이드

## 1단계: Supabase 프로젝트 생성 (5분)

### 1-1. Supabase 가입 및 프로젝트 생성

1. 🌐 https://supabase.com 접속
2. "Start your project" 또는 "Sign in" 클릭
3. GitHub 계정으로 로그인 (또는 이메일 가입)
4. "New Project" 클릭
5. 다음 정보 입력:
   - **Name**: `bubu-gagyebu` (또는 원하는 이름)
   - **Database Password**: 강력한 비밀번호 입력 (꼭 기억하세요!)
   - **Region**: `Northeast Asia (Seoul)` 선택
   - **Pricing Plan**: `Free` 선택
6. "Create new project" 클릭 (약 2분 소요)

### 1-2. 환경변수 복사

프로젝트 생성이 완료되면:

1. 왼쪽 사이드바 하단의 ⚙️ **"Settings"** 클릭
2. **"API"** 메뉴 클릭
3. 📋 다음 정보를 복사해두세요:
   - **Project URL** 
   - **anon public** key (긴 문자열)

---

## 2단계: 환경변수 설정 (1분)

### 프로젝트 루트에 `.env.local` 파일 생성

1. 프로젝트 폴더에서 `.env.local` 파일을 새로 만듭니다
2. 다음 내용을 복사하여 붙여넣고, 값을 수정하세요:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://여기에프로젝트URL입력.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=여기에anon_public_key입력
```

**예시:**
```bash
NEXT_PUBLIC_SUPABASE_URL=https://abcdefghijk.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprIiwicm9sZSI6ImFub24iLCJpYXQiOjE2ODc1MjkyMDAsImV4cCI6MjAwMzEwNTIwMH0.xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

⚠️ **주의**: 
- 따옴표 없이 입력
- 공백 없이 입력
- 파일명 정확히: `.env.local` (앞에 점 있음)

## 3단계: 데이터베이스 테이블 생성 (2분)

### 3-1. SQL Editor 열기

1. Supabase 대시보드로 돌아가기
2. 왼쪽 사이드바에서 **"SQL Editor"** 클릭
3. "+ New query" 클릭

### 3-2. SQL 스크립트 실행

1. `supabase/schema.sql` 파일 열기 (프로젝트 폴더에서)
2. 파일 내용 **전체 복사** (Ctrl+A → Ctrl+C)
3. Supabase SQL Editor에 **붙여넣기** (Ctrl+V)
4. 오른쪽 하단 **"Run"** 버튼 클릭 (또는 Ctrl+Enter)
5. ✅ "Success. No rows returned" 메시지 확인

## 🎉 사용 시작!

### 기본 사용법

1. **거래 추가**: 오른쪽 하단 **+ 버튼** 클릭
2. **예산 관리**: 하단 탭에서 **"예산"** 클릭
3. **할일 공유**: 하단 탭에서 **"할일"** 클릭
4. **링크 공유**: 상단 **📋 복사 버튼**으로 가족과 공유

### 금액 입력 방법
- `10` 입력 = 10만원
- `100` 입력 = 100만원
- `0.5` 같은 소수는 입력 불가 (1만원 단위만)

## ❓ 문제 해결

### "supabaseUrl is required" 에러
- `.env.local` 파일이 프로젝트 **루트 폴더**에 있는지 확인
- 파일명이 정확한지 확인 (`.env.local`)
- 환경변수에 **공백이나 따옴표**가 없는지 확인
- 개발 서버 **재시작** (Ctrl+C 후 `npm run dev`)

### 데이터가 저장되지 않음
- Supabase에서 SQL 스크립트를 실행했는지 확인
- Supabase 대시보드 → **"Table Editor"**에서 테이블 확인:
  - household
  - transactions
  - budget_items
  - todos
  - monthly_balances

### 연결 오류
- Supabase 프로젝트가 **활성 상태**인지 확인 (대시보드에서)
- 방화벽이나 VPN으로 인한 차단 확인
- 브라우저 콘솔(F12)에서 에러 메시지 확인
